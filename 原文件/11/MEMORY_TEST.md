### 2026-02-12 - 网络获取工具优先级与备用原则（最高优先级）
- **决策**：强制使用 agent-browser CLI 工具，任何网络获取任务必须遵循此原则
- **核心原则**：
  - **优先级**：无条件、强制优先通过 `agent-browser` CLI 工具完成
  - **涵盖范围**：网页内容读取、搜索引擎查询、GitHub 仓库文件访问等
  - **工具说明**：`agent-browser` 是基于 Playwright 的命令行浏览器自动化工具，用于模拟浏览器行为以执行网页获取、搜索及复杂交互任务

- **操作规范**：
  1. **环境配置与启动**：
     - 优先通过 `--executable-path` 指定系统浏览器路径
     - 或运行 `agent-browser install`
     - **重要**：通过 Bash 调用时，需设置环境变量：
       ```bash
       export AGENT_BROWSER_EXECUTABLE_PATH="C:\Users\Administrator\AppData\Local\Google\Chrome\Application\chrome.exe"
       ```
  
  2. **会话管理**：
     - 在启动新任务前，务必使用 `agent-browser close` 关闭任何旧会话以避免冲突
  
  3. **标准工作流**（强制要求）：
     ```
     open <URL> -> wait 等待加载 -> snapshot 获取页面结构
     ```
     - **始终**利用 `snapshot` 获取元素引用以提高操作稳定性
     - 再利用 `click`, `fill`, `scroll` 等命令进行精确交互
     - 最终以 `close` 释放资源

- **调用触发条件**：
  - **强制优先使用 agent-browser，除非**：
     - 用户明确指定使用 MCP web-reader、MCP web-search-prime 或 MCP zread

- **备用工具调用流程**：
  - **触发条件**（满足任一）：
     1. `agent-browser` 结果不满意
     2. `agent-browser` 获取信息失败
     3. 结果未满足用户预期
     4. 用户明确指定
  
  - **执行流程**：
     1. 立即暂停
     2. 主动询问用户：
        > "agent-browser 的结果似乎不尽如人意/用户已明确指定，您是否希望尝试使用 [指定备用工具，例如：MCP web-reader / MCP web-search-prime / MCP zread] 来完成此任务？"
     3. **必须**得到用户明确同意后方可调用备用工具

- **禁止事项**：
  - 在其他任何情况下，严禁调用除 `agent-browser` 之外的网络获取工具
